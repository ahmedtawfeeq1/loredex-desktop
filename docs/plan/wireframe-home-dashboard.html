Home becomes a directional dashboard, not a document

The Home view stops rendering the Start-Here brief as a narrow page of prose and becomes a full-width insight dashboard: what needs attention, who blocks whom, where contracts churn, and how fresh each project's knowledge is — every tile a one-click jump into the view that acts on it. Zero new backend asks: every number maps to an existing core-host channel (dashboard.build alone carries per-project state, open handoffs with ageDays, and flow edges).

?Open questions (product)

Q1 — Age severity thresholds? Attention chips amber at ≥2 days, rust at ≥5? Safe default: 2/5, configurable later in Settings. Affects only chip color, not ranking.
Q2 — Company-wide or "my projects" first? Identity carries no project membership today, so v1 is company-wide with a project filter chip row. Per-user scoping would need a membership concept (vault or app-db) — deferred unless you want it now.
Q3 — Churn window. 7 days fixed, or follow a Settings value shared with the Contracts view? Default: fixed 7d, revisit with usage.
Q4 — Does the rendered brief survive anywhere on Home? Current plan: link-out card only (the Reader owns prose). Alternative: collapsible preview under the brief card. Default: link-out only — keeps the dashboard a dashboard.
